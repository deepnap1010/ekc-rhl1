# Deploying EKC SmartFactory (GitHub ‚Üí Render)

This app is hosted as **one Render Web Service**: the Node/Express server serves the
API + socket.io **and** the built React client from a single URL. No CORS, no second
host, no client config ‚Äî the client already calls `/api/v1` and `socket.io` relative
to its own origin.

## Prerequisites
- A **GitHub** account.
- A **Render** account (https://render.com ‚Äî sign up with GitHub).
- Your **MongoDB Atlas** connection string (the existing EKC cluster).

---

## 1 ‚Äî Push the code to GitHub

The repo is already initialized and committed locally (secrets are excluded by
`.gitignore` ‚Äî `.env` files are never committed). Create an empty repo on GitHub
(no README/license), then from `D:\ekc-rhl`:

```bash
git remote add origin https://github.com/<you>/ekc-smartfactory.git
git branch -M main
git push -u origin main
```

> If you used the GitHub CLI: `gh repo create ekc-smartfactory --private --source=. --push`

---

## 2 ‚Äî Create the Render service from the Blueprint

1. Render dashboard ‚Üí **New +** ‚Üí **Blueprint**.
2. Connect your GitHub and pick the `ekc-smartfactory` repo.
3. Render reads **`render.yaml`** and proposes the `ekc-smartfactory` web service. Click **Apply**.

It will use:
- **Build:** `cd client && npm install --include=dev && npm run build && cd ../server && npm install`
- **Start:** `cd server && npm start`
- **Health check:** `/health`

---

## 3 ‚Äî Set the secrets

In the service's **Environment** tab, set the values marked `sync: false`:

| Key             | Value                                                            |
|-----------------|------------------------------------------------------------------|
| `MONGO_URI`     | your Atlas connection string (e.g. `mongodb+srv://user:pass@cluster.../`) |
| `ADMIN_EMAIL`   | *(optional)* only used if the `users` collection is empty        |
| `ADMIN_PASSWORD`| *(optional)* bootstrap password for the above                    |

Already handled by the Blueprint (no action needed):
- `JWT_SECRET` ‚Äî auto-generated by Render.
- `NODE_ENV=production`, `DB_NAME=test`.
- `PORT` ‚Äî injected by Render automatically; the server reads it.

> Since your Atlas already has real users, the bootstrap admin stays disabled and you
> log in with your existing credentials (e.g. `admin@ekc.in`).

---

## 4 ‚Äî Allow Render to reach MongoDB Atlas

Atlas ‚Üí **Network Access** ‚Üí **Add IP Address** ‚Üí **Allow access from anywhere**
(`0.0.0.0/0`). (Render's free tier doesn't give static egress IPs, so this is the
simplest option. For a paid tier you can pin Render's static outbound IPs instead.)

---

## 5 ‚Äî Deploy & verify

- Render builds and starts automatically. Watch the **Logs** for
  `[server] EKC SmartFactory API on :10000 (production)`.
- Open the service URL (e.g. `https://ekc-smartfactory.onrender.com`) ‚Üí the login page loads.
- Sanity check the API directly: `https://<your-url>/health` ‚Üí `{"ok":true,...}`.
- Log in and confirm the dashboard + live machine data appear.

---

## Day-to-day

- **Redeploy:** just `git push` to `main` ‚Äî Render auto-deploys (`autoDeploy: true`).
- **Free-tier sleep:** the service sleeps after ~15 min idle; the next request wakes it
  (a ~30‚Äì50 s cold start). Upgrade to a paid instance to keep it always-on.
- **Region:** `render.yaml` uses `singapore` (closest to India). Change `region:` and
  redeploy to move it (e.g. `oregon`, `frankfurt`).
- **Custom domain:** add it under the service's **Settings ‚Üí Custom Domains**.

## Local development (unchanged)

```bash
cd server && npm install && npm run dev     # API on :5000 (needs server/.env)
cd client && npm install && npm run dev     # UI on :5173, proxies to :5000
```
